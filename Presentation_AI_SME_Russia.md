# СЛАЙД 7: Практические аспекты данных и ИТ-систем

## Данные — ключевой актив для ИИ

### Требования к «ИИ-годным» данным:
- 📊 Достаточный объём исторических записей (1-2 года минимум)
- 🧹 Очистка от грубых ошибок и дубликатов
- 🏷️ Разметка при необходимости
- 🔄 Непрерывный доступ к актуальным данным

### Типовая системная среда МСБ (Россия):
| Система | Назначение |
|---------|------------|
| **1С** | Учёт и ERP |
| **Bitrix24** | CRM и коммуникации |
| **IP-АТС** | Телефония, колл-центр |
| **Excel** | Различные потребности |

### Варианты интеграции:
1. **API** — прямое подключение (1С, Bitrix24 поддерживают REST API)
2. **ETL** — регулярные выгрузки (CSV, JSON)
3. **RPA** — роботизация интерфейса (крайний случай)

> **Принцип:** Для пилота достаточно минимальной интеграции

# СЛАЙД 12: Логика расчёта ROI

## Структура экономического обоснования

### Выгоды (Benefits):

| Категория | Описание |
|-----------|----------|
| **Экономия времени** | Автоматизация → высвобождение часов → ₽ через стоимость человеко-часа |
| **Снижение ошибок** | Меньше переделок, штрафов, возвратов |
| **Увеличение производительности** | Больше заявок обработано в единицу времени |
| **Рост доходов** | Рекомендации → доп. покупки, скоринг → больше сделок |
| **Сокращение рисков** | Предотвращение аварий, мошенничества, утечек |

### Затраты (Costs):

| Категория | Описание |
|-----------|----------|
| **ПО и инфраструктура** | Лицензии на облачные сервисы (SaaS), облачные API, серверы |
| **Разработка и интеграция** | Труд команды, интеграция с системами |
| **Подготовка данных** | Сбор, очистка, разметка (до 80% усилий!) |
| **Обучение** | Тренинги, адаптация персонала |
| **Эксплуатация (Opex)** | Поддержка, мониторинг, затраты на вычисления |

### Формула ROI:
```
ROI = (Годовой бенефит - Затраты) / Затраты × 100%
```

# СЛАЙД 14: Облачные сервисы (SaaS) vs. заказная разработка

## Рекомендации по выбору подхода

### Сравнение подходов:

| Критерий | SaaS (Облако) | Гибридный | Заказной (Custom) |
|----------|------|--------|--------|
| **Срок выхода (Time-to-Value)** | ⚡ Дни | ⏱️ Недели | 🐢 Месяцы |
| **Стоимость старта** | 💰 Низкая | 💰💰 Средняя | 💰💰💰 Высокая |
| **Кастомизация** | ❌ Ограничена | ✅ Частичная | ✅✅ Полная |
| **Контроль данных** | ⚠️ У вендора | ✅ Частичный | ✅✅ Полный |
| **Требования к команде** | 👤 Минимум | 👥 Средние | 👥👥 Высокие |

### Когда что выбрать:

| Сценарий | Рекомендация |
|----------|--------------|
| Быстро показать результат | **SaaS (Готовое решение)** |
| Чувствительные данные (ПДн) | **Локальные / Гибридные решения** |
| Глубокая интеграция | **Заказная разработка (Custom)** |
| Высокая нагрузка | **Заказная разработка (экономия в долгосрочной перспективе)** |
| Риск привязки к поставщику (Vendor lock-in) | **Open-source / Гибрид** |

### 💡 Рекомендация для МСБ:
> **Начните с SaaS или low-code решений → докажите ROI → переходите к гибридной схеме при росте требований**

# СЛАЙД 15: Дорожная карта (Roadmap) — Фазы внедрения

## Дорожная карта проекта

### Фаза 0: Исследование (Discovery) (1-2 недели)
- **Цели:** Выявить сценарии использования, оценить готовность, сформировать бизнес-кейс
- **Результаты (Deliverables):** Список идей, отчет о готовности, дорожная карта, план пилота
- **Завершение:** Документ «Видение и план» утверждён

### Фаза 1: Пилот / Проверка концепции (PoC) (2-6 недель)
- **Цели:** Прототип на выбранном кейсе, проверка ценности
- **Результаты:** Рабочий прототип, метрики качества, отчёт о пилоте
- **Завершение:** Решение о переходе к MVP

### Фаза 2: MVP и развертывание (4-12 недель)
- **Цели:** Продуктивная версия, интеграция, обучение пользователей
- **Результаты:** Рабочий сервис, документация, обученные пользователи
- **Завершение:** MVP стабильно работает, первые подтверждённые выгоды

### Фаза 3: Масштабирование и эксплуатация (постоянно)
- **Цели:** Широкое внедрение, мониторинг, постоянное улучшение
- **Результаты:** Масштабирование, панель метрик, обновления

# СЛАЙД 16: Процесс реализации (Delivery Workflow) — 10 шагов

## Сквозной процесс поставки решения

```
1. ПРИЕМ (INTAKE)  → Сбор идей использования ИИ
        ↓
2. ОЦЕНКА          → Оценка данных, выгоды, сложности
        ↓
3. ПРИОРИТИЗАЦИЯ   → Согласование с бизнесом
        ↓
4. ДОСТУП К ДАННЫМ → Выгрузки, очистка, датасет
        ↓
5. ПРОЕКТИРОВАНИЕ  → Архитектура, техзадание
        ↓
6. РАЗРАБОТКА      → Создание / настройка модели
        ↓
7. ТЕСТИРОВАНИЕ    → Тестирование на проверочных данных
        ↓
8. ПРОВЕРКА БЕЗОПАСНОСТИ → Ревью кода, ПДн, комплаенс
        ↓
9. ЗАПУСК (GO-LIVE) → Развёртывание, обучение, мягкий запуск
        ↓
10. МОНИТОРИНГ     → Метрики, фидбэк, итерации
```

### Ключевые контрольные точки:
- 📍 Между 2-3: Правильный выбор фокуса
- 📍 Между 7-8: Не запускать без проверки качества и безопасности
- 📍 Между 9-10: Не бросать после релиза — измерять и улучшать

# СЛАЙД 17: Контрольные точки качества (Quality Gates)

## Минимум 10 обязательных проверок

| # | Контрольная точка (Quality Gate) | Описание |
|---|--------------|----------|
| 1 | **Бизнес-KPI и базовые показатели (baseline)** | Метрики успеха согласованы, текущие значения измерены |

# СЛАЙД 31: Анализ модели SaaS (Плюсы и минусы для МСБ)

## Что выбрать: Облако или Свой сервер?

### ✅ Преимущества SaaS (Облако):
1. **Быстрый старт:** Регистрация и начало работы за 1 день.
2. **Низкий порог входа:** Оплата помесячно (OpEx), нет крупных вложений.
3. **Авто-обновления:** Всегда актуальная версия ПО.
4. **Масштабируемость:** Легко добавить лицензии при росте штата.

### ❌ Недостатки и Риски SaaS:
1. **Приватность:** Данные хранятся у вендора (риск утечек).
2. **Vendor Lock-in:** Сложно переехать, если вендор поднимет цены.
3. **Кастомизация:** Только то, что разрешил разработчик.

### 🇷🇺 Особенности в России:
- **Риск блокировок:** Западные SaaS могут отключить аккаунт.
- **152-ФЗ:** Использование зарубежных SaaS для ПДн ограничено.
- **Рекомендация:** Выбирать **Российские SaaS** (в реестре ПО) или **Self-hosted** решения (Bitrix24 Box, Open Source) для критичных данных.
