1. Executive Summary

This artifact defines the AI-first integration layer required to bridge legacy systems (Bitrix24, SAP) with modern LLM workflows. The architecture relies heavily on dynamically provisioned Model Context Protocol (MCP) servers as a bridge.

2. Component Topology

DACH Market Integrations

Native configuration using Open-Source standard MCP definitions to ensure maximum compliance and API depth.

  • Salesforce: @mcp-server-salesforce
  • HubSpot: @shinzo-labs/hubspot-mcp
PRODUCTION READY

Generic Integrations (Composio)

Fallback for wide-net systems like Trello or regional platforms without native MCP servers.

  • Action execution via universal REST proxy
  • Configuration driven auth management

CIS / RU Market Legacy

Connecting to isolated instances (1C, Bitrix24 On-Prem) requires specific tunnel configurations.

  • Custom XML-RPC wrappers required
  • Security audit pending for reverse-proxies
REQUIRES CUSTOM AGENT

3. Configuration Blueprint

Standardized JSON configuration deployed via the Plugin Architecture to initialize the session context. 

{
  "mcpServers": {
    "salesforce": {
      "command": "npx",
      "args": ["-y", "@mcp-server/salesforce"],
      "env": {
        "SF_INSTANCE_URL": "${SF_ENV_URL}",
        "SF_API_VERSION": "60.0"
      }
    }
  }
}
